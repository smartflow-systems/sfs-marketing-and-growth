## Common Patterns Across ALL SFS Projects

### CI/CD Standards
- GitHub Actions workflows: `.github/workflows/ci.yml`, `.github/workflows/sfs-ci-deploy.yml`
- Reusable SFS CI patterns
- Health check endpoints: `GET /health ‚Üí {"ok":true}`
- Badge display in README files

### Deployment Strategy
- **Platform:** Replit (primary)
- **Required Env Vars:** `SFS_PAT`, `REPLIT_TOKEN`, `SFS_SYNC_URL`
- **Port Config:** `${PORT:-5000}` or app-specific defaults

### Theme & Design (CRITICAL)
- **Signature Colors:** Brown/black/gold palette
- **UI Libraries:** Tailwind CSS, Radix UI components
- **Typography:** Consistent font systems across projects
- **Always maintain brown/black/gold aesthetic in all SFS projects**

### Authentication Patterns
- JWT token-based authentication
- Role-based access control: Admin, Staff, Analyst, Owner
- Invitation systems with expiration tokens

### Database Strategies
- **ORM:** Prisma (primary)
- **Dev:** SQLite
- **Production:** PostgreSQL
- **Migrations:** `prisma migrate dev`

### AI Integration Standards
- **Provider:** OpenAI GPT-4 (primary)
- **Orchestration:** LangChain
- **Vector DB:** ChromaDB for embeddings

### Payment Processing
- **Processor:** Stripe (primary)
- **Multi-tenant:** Stripe Connect
- **Models:** Freemium, tier-based pricing, usage-based billing

## Development Workflow Standards

### Branch Strategy
- `main` ‚Üí production-ready code
- Feature branches for development
- **GitHub = source of truth**

### Agent Notes Pattern
- All repos include `AGENTS.md` for AI assistant guidance
- Show file paths in brackets: `[path/to/file.ts]`
- VERIFY + UNDO before destructive operations
- Bash scripts use `set -euo pipefail`

### Health Checks
- All deployable apps: `/health` endpoint
- Returns: `{"ok":true}`
- Script location: `scripts/health.sh`
